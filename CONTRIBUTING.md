# Contributing

Thanks for your interest in contributing to this project. Whether you're reporting a bug, suggesting an improvement, or submitting a fix, your input is appreciated.

## Reporting Bugs

If you find a bug, please [open an issue](https://github.com/JoshDoesIT/JoshDoes.IT-Website/issues/new) with the following information:

- A clear description of the problem
- Steps to reproduce the behavior
- Expected vs. actual behavior
- Browser and OS details, if relevant

For security vulnerabilities, please refer to [SECURITY.md](SECURITY.md) instead of opening a public issue.

## Suggesting Changes

Before starting work on a change, please open an issue to discuss the idea first. This helps avoid duplicate effort and ensures the change aligns with the project's direction.

## Pull Requests

1. Fork the repository and create your branch from `main`
2. Install dependencies with `npm install`
3. Make your changes, following the existing code style
4. Add or update tests to cover your changes
5. Run `npm run test:run` (unit tests) and `npm run test:e2e` (E2E tests) to verify everything passes
6. Run `npm run lint` to check for linting issues
7. Submit your pull request with a clear description of the change

## Development Setup

See the [README](README.md#local-development) for instructions on getting the project running locally.

## Code Style

This project uses ESLint and Prettier to enforce consistent formatting. Please follow the existing patterns in the codebase and ensure your changes pass linting before submitting.
